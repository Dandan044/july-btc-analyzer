# 警报调试报告任务

当收到"警报调试报告"指令时，**必须按顺序完成以下步骤**：

## ⚠️ 重要：必须保存报告文件

本任务的核心目标是生成并保存报告文件，**不可跳过此步骤**。

---

## 执行步骤

### 第1步：解析输入数据

从任务指令中提取警报触发时收集的数据，包括：
- 警报名称
- 触发时间
- 价格/K线数据
- 其他市场数据

### 第2步：生成报告内容（必须执行）

根据数据类型生成报告内容：

**如果有K线数据**，使用表格格式：
```markdown
# 🔔 警报触发报告

**警报名称**: xxx  
**触发时间**: YYYY-MM-DD HH:mm:ss  
**警报类型**: xxx

---

## 📊 K线数据

| 时间 | 开盘 | 最高 | 最低 | 收盘 | 交易量 |
|------|------|------|------|------|--------|
| ... | ... | ... | ... | ... | ... |

---

## 📋 完整数据

\`\`\`json
{原始JSON数据}
\`\`\`

---

*报告生成时间: YYYY-MM-DD HH:mm:ss*
```

**如果是价格数据**，使用以下格式：
```markdown
# 🔔 警报触发报告

**警报名称**: xxx  
**触发时间**: YYYY-MM-DD HH:mm:ss  
**警报类型**: xxx

---

## 💰 价格信息

| 项目 | 数值 |
|------|------|
| 当前价格 | $xxx |
| 24h最高 | $xxx |
| 24h最低 | $xxx |
| 1h涨跌 | x% |
| 24h涨跌 | x% |
| 24h交易量 | xxx |

---

## 📋 完整数据

\`\`\`json
{原始JSON数据}
\`\`\`

---

*报告生成时间: YYYY-MM-DD HH:mm:ss*
```

### 第3步：保存报告文件（必须执行）

**使用 write 工具**将报告保存到以下路径：
```
/root/.openclaw/workspace-july/reports/alert-report-YYYY-MM-DD-HHMM.md
```

文件名中的时间使用触发时间生成，格式如：`alert-report-2026-03-04-1531.md`

**必须确认文件已成功保存**。

### 第4步：发送到飞书

将生成的文件发送至飞书：'ou_4b65a3a145ee00ae60ae2283a839f46c'


### 第5步：确认完成

输出确认信息：
- ✅ 报告已保存：`reports/alert-report-xxx.md`
- ✅ 飞书发送状态

---

执行前确认：

- [ ] 已生成报告内容
- [ ] 已使用 write 工具保存文件
- [ ] 已发送到飞书
